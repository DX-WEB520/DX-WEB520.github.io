# 什么是 CSS 预处理器

CSS 预处理器定义了一种新的语言，用一种专门的编程语言，为 CSS 增加了一些编程的特性，（例如你可以在 CSS 中使用变量、简单的逻辑程序、函数等等）无需考虑浏览器的兼容性问题，将 CSS 作为目标生成文件，然后开发者就只要使用这种语言进行编码工作。

# CSS 预处理器的特点

  • 基于 CSS 的另一种语言
  • 通过工具编译成 CSS
  • 添加了很多 CSS 不具备的特性
  • 更好的组织 CSS

# CSS 预处理器优缺点

  1. 优点
  • CSS 更加简洁
  • 适应性更强
  • 可读性更佳
  • 更易于代码的维护
  1. 缺点
  • 需要引入编程过程，有学习成本

# CSS 预处理器的能力

  • 变量和计算 减少重复代码
  • 嵌套 反映层级和约束
  • Extend 和 Mixin 复用代码片段
  • 循环 适应于复杂有规律的样式
  • import CSS 文件模块化

# CSS 预处理器的语言

  • Sass(.scss)
  • Less(.less)
  • Stylus
  • DT CSS
  • Swithch CSS

# Less

# less 简介

css 预处理语言，扩展了 css 语言，增加了变量、Mixin、函数等特性，使 css 更易维护和扩展，可运行在 Node 或浏览器端。

# less 安装

  • npm install -g less

# less 的功能

  • 变量 (variables)：符号 @
  • 混合 (Mixins)
  • 嵌套 (Nesting)
  • 扩展 (Extend)
  • 作用域 (Scope)
  • 循环 (Loop)
  • 导入 (Importing)

# less 的编译

在 nodejs 环境中使用 less

  • 在命令行中运行：lessc styles.less styles.css
  • 编译成生产环境下的 css 文件：lessc -x styles.less styles.css

    在浏览器环境中使用 less

# Sass

# Sass 简介

Sass 是一款强化 CSS 的辅助工具，采用 Ruby 语言编写，配合 HAML（一种缩进式 HTML 预编译器）而设计的，有着和 HTML 一样的缩进式风格。后缀为.scss

# Sass 的编译

  • 命令编译

    • 单文件编译：sass styles.scss styles.css
    • 实时监视.scss 文件：sass --watch styles.scss:styles.css

  • GUI 界面工具编译

    • 考拉工具 Koala
    • CodeKit

  • 自动化编译 (利用 Grunt 和 Gulp)

# Sass 的功能

  • sass 变量：符号 $，默认变量！default, 局部变量和全局变量！global
  • sass 嵌套：选择器嵌套、属性嵌套、伪类嵌套
  • sass mixin 混入：@mixin 声明 /@include 调用
  • sass extend 扩展：@extend
  • sass loop 循环
  • sass import 模块化
